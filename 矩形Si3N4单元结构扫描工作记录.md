# 矩形 Si3N4 单元结构扫描工作记录

## 1. 当前目标

当前工作目标是以矩形 Si3N4 超表面单元为基础，建立一个可用于后续 PINN 数据库和随机加密超表面设计的 FDTD 仿真样例。

这一阶段重点不是完整数据库，而是先验证：

- 单元结构是否设置合理；
- 扫描脚本能否自动修改结构并批量仿真；
- 不同结构尺寸下的透射率、反射率和相位响应是否有可用规律；
- 是否有适合作为随机加密候选波段的区域。

## 2. 当前 FDTD 模型

当前模型文件：

```text
rect_sin_unit_cell.fsp
```

模型结构：

```text
Si3N4 矩形柱 / SiO2 基底
```

主要参数：

```text
周期 period = 300 nm
Si3N4 高度 = 800 nm
基底材料 = SiO2 (Glass) - Palik
柱子材料 = Si3N4 (Silicon Nitride) - Luke
边界条件 = x/y Periodic, z PML
波长范围 = 400-700 nm
波长点数 = 101
偏振 = 线偏振 Ex, 0 deg
```

当前基准结构：

```text
rect_lx = 120 nm
rect_ly = 200 nm
```

z 方向布局：

```text
Si3N4: z = 0 到 800 nm
SiO2: z = -800 到 0 nm
FDTD: z = -500 到 2000 nm
source: z = 1550 nm，向 -z 方向入射
R monitor: z = 1200 nm
T monitor: z = -300 nm
```

说明：

- FDTD 下边界位于 SiO2 内部；
- 反射监视器位于光源和结构之间；
- 反射监视器距离 Si3N4 顶部约 400 nm，满足大于 300 nm 的要求；
- 透射监视器位于 SiO2 基底内部。

## 3. 当前脚本

### 3.1 建模脚本

```text
rect_sin_unit_cell_setup.lsf
```

作用：

- 创建矩形 Si3N4 单元结构；
- 设置 SiO2 基底；
- 设置 FDTD 区域、光源和监视器；
- 保存 `rect_sin_unit_cell.fsp`。

当前材料已经同步为：

```text
Si3N4 (Silicon Nitride) - Luke
SiO2 (Glass) - Palik
```

### 3.2 完整导出脚本

```text
rect_sin_unit_cell_export.lsf
```

作用：

- 导出完整光谱、相位和场数据；
- 文件较大，不适合批量扫描。

之前完整场文件大小约为：

```text
978 MB
```

### 3.3 峰值场导出脚本

```text
rect_sin_unit_cell_export_peak_fields.lsf
```

作用：

- 保存完整 R/T/相位光谱；
- 只保存透射峰值附近 3 个波长点的场数据；
- 文件大小约为 29 MB。

后续计划：

- 将 3D 光场改为 2D 截面光场；
- 建议使用 x-z 截面，例如 `nearfield_xz`；
- 这样可以进一步减小文件，并且仍能观察结构附近场分布。

### 3.4 50 点结构扫描脚本

```text
rect_sin_unit_cell_sweep_50_light.lsf
```

作用：

- 加载同一个 `rect_sin_unit_cell.fsp`；
- 在脚本中循环修改 `pillar_rect` 的 `x span` 和 `y span`；
- 每个尺寸运行一次 FDTD；
- 提取 R/T/相位；
- 将 50 组结果统一保存到一个 `.mat` 文件。

核心逻辑：

```lsf
switchtolayout;
setnamed("pillar_rect","x span",lx_nm(i)*nm);
setnamed("pillar_rect","y span",ly_nm(i)*nm);
run;
```

说明：

- 扫描不生成 50 个 `.fsp` 文件；
- 只使用一个 `.fsp` 模板；
- 50 组结构的结果保存在一个结果文件中。

扫描输出文件：

```text
rect_sin_unit_cell_sweep_50_light.mat
```

结果文件大小约：

```text
297 KB
```

## 4. 50 组结构扫描方案

总扫描点数：

```text
50
```

分为两类：

```text
35 个：长宽等比例缩放
15 个：长宽单独变化
```

### 4.1 长宽等比例缩放

基准结构：

```text
rect_lx = 120 nm
rect_ly = 200 nm
```

缩放方式：

```text
rect_lx = 120 * s
rect_ly = 200 * s
```

扫描范围：

```text
s = 0.50 : 0.02 : 1.18
```

对应范围：

```text
rect_lx = 60 到 141.6 nm
rect_ly = 100 到 236 nm
```

### 4.2 长宽单独变化

固定 `rect_ly = 200 nm`，改变 `rect_lx`：

```text
(60, 200)
(80, 200)
(100, 200)
(140, 200)
(160, 200)
(180, 200)
```

固定 `rect_lx = 120 nm`，改变 `rect_ly`：

```text
(120, 80)
(120, 120)
(120, 160)
(120, 220)
(120, 240)
```

额外补充结构：

```text
(80, 240)
(100, 240)
(160, 120)
(200, 120)
```

## 5. 50 组扫描结果

扫描完成情况：

```text
last_completed = 50
```

全局结果：

```text
T_spectrum 范围 = 0.248209 到 1.00434
R_spectrum 范围 = 2.94e-05 到 0.752504
R + T 范围 = 0.619294 到 1.02881
```

说明：

- 大多数结构透射率较高；
- 少数大尺寸结构出现强反射或能量偏差；
- 当前结果适合做初筛，但不应将所有结构直接作为最终数据库点。

## 8. 全 50 结构圆周相位覆盖

为了判断某个波长下是否有较好的相位调控能力，采用圆周相位覆盖方法：

```text
固定一个波长；
取 50 个结构的 phase_T；
将相位放到 0-2π 圆周上；
计算覆盖范围。
```

代表波长结果：

```text
430 nm: 全 50 结构覆盖约 0.951 × 2π
451 nm: 全 50 结构覆盖约 0.842 × 2π
475 nm: 全 50 结构覆盖约 0.770 × 2π
499 nm: 全 50 结构覆盖约 0.791 × 2π
526 nm: 全 50 结构覆盖约 0.691 × 2π
550 nm: 全 50 结构覆盖约 0.623 × 2π
574 nm: 全 50 结构覆盖约 0.567 × 2π
601 nm: 全 50 结构覆盖约 0.514 × 2π
625 nm: 全 50 结构覆盖约 0.474 × 2π

700 nm: 全 50 结构覆盖约 0.385 × 2π
```

在 `T > 0.95` 条件下，较好的波长包括：

```text

```

结论：

- 当前 50 个矩形结构在 400-430 nm 附近相位覆盖最好；
- 450-500 nm 仍然有较好的相位覆盖潜力；
- 480-490 nm 是透射率和相位覆盖综合较好的区域；
- 500-700 nm 虽然透射率较高，但相位覆盖逐渐不足。

## 9. 对随机加密应用的判断

如果用于传统超透镜或精确波前重构，通常希望实现完整 `0-2π` 相位覆盖。

但如果用于随机加密，不一定必须完整覆盖 `2π`。随机加密更关注：

- 不同结构输出是否有足够差异；
- 相位和场分布是否具有可区分性；
- 透射率是否足够高；
- 输出是否稳定、可重复；
- 是否能形成可用于加密/解密的随机映射。

因此当前 `480-490 nm` 附近虽然没有完整 `2π`，但具有：

```text
较高透射率；
较大相位覆盖；
结构响应较稳定。
```

这个波段可以作为随机加密方向的优先候选波段。

## 10. 下一步建议

### 10.1 第二轮结构扫描

建议围绕表现较好的区域继续扫描：

```text
lx = 80-130 nm
ly = 80-220 nm
```

重点关注：

```text
(120,120)
(120,80)
(100,200)
(80,200)
(80,240)
(100,240)
```

### 10.2 波长重点

建议优先分析：

```text
480-490 nm
```

原因：

- 透射率整体较高；
- 相位覆盖接近 `0.85 × 2π`；
- 比 400 nm 附近更稳定；
- 对随机加密应用有潜力。

### 10.3 光场数据

第一轮扫描只保存光谱和相位，不保存大体积光场。

后续建议：

- 对筛选出的少数代表结构保存光场；
- 将 3D 光场改为 2D 截面光场；
- 保存峰值波长或目标波长附近的场分布；
- 不对所有扫描结构保存完整 3D 场。

### 10.4 后续数据库设计

当前结果可作为数据库设计依据：

- 矩形结构可作为第一类结构；
- 不建议盲目扩大尺寸到强反射区域；
- 后续可以加入圆形、平行四边形、H 形、十字形；
- 随机加密方向不必强制所有结构完整 `2π` 覆盖；
- 更应关注结构响应差异、透射稳定性和输出光场随机性。
